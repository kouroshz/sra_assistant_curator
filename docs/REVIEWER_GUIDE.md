# Reviewer Guide

This guide is for reviewers, collaborators, and lab members who want to understand and validate the SRA Paper Curator repository.

---

## Purpose

This repository provides a curator-assist workflow for transforming public RNA-seq and ChIP-seq metadata, together with paper context, into human-reviewable curation workbooks.

The current motivating use case is public Plasmodium sequencing metadata curation for downstream gene-regulatory-network analysis.

The workflow is not intended to replace human curation. It is intended to reduce curator burden while preserving provenance, validation, and auditability.

---

## What this repository contains

The repository contains:

- active RNA and ChIP workflow scripts
- a production-facing workflow wrapper
- deterministic public metadata enrichment scripts
- optional AI-assisted packet review scripts
- deterministic validation and QC scripts
- final curator release packaging scripts
- golden-output regression tests
- documentation for curators and developers
- archived legacy and prototype scripts

---

## What this repository does not contain

The repository intentionally does not contain:

- generated outputs
- generated results
- raw PDFs
- raw AI JSON files
- .env files
- API keys
- bulky intermediate artifacts

These are generated locally and ignored by Git.

## Installation

Create the conda environment:

    conda env create -f environment.yml
    conda activate sra_paper_curator

The environment uses Python 3.11 and includes packages for table handling, Excel generation, YAML/config handling, PDF parsing, and optional OpenAI/API use.

---

## Fresh-clone validation

From a fresh clone:

    python scripts/05_run_all_checks.py
    python scripts/04_pipeline_readiness_report.py

Expected behavior:

- Python files compile
- workflow wrapper runs in dry-run mode
- AI-capable steps refuse execution without explicit opt-in
- artifact-backed checks are skipped because generated outputs are absent
- final verdict is PASS for repo-level smoke checks

This is the appropriate first validation for reviewers.

---

## Artifact-backed validation

On a production or workstation checkout that already contains generated outputs:

    python scripts/05_run_all_checks.py --with-artifacts
    python scripts/04_pipeline_readiness_report.py

Expected behavior:

- final curator release is rebuilt
- release QC passes
- golden-output regression tests pass
- workflow dry-run behavior passes
- AI safety guard passes

## Workflow map

The active workflow is documented in:

    docs/ACTIVE_WORKFLOW_MAP.md
    workflows/steps.tsv

List workflow steps:

    python workflows/run_workflow_step.py --list

Show one step without running it:

    python workflows/run_workflow_step.py --step 90

All workflow steps are dry-run by default.

---

## AI/API safety

AI-assisted steps are optional.

They require all of:

    --execute
    --execute-ai
    AGENTIC_AI_ENABLE_API=1

Example:

    AGENTIC_AI_ENABLE_API=1 python workflows/run_workflow_step.py --step 33 --execute --execute-ai

Without explicit AI opt-in, the wrapper refuses to execute AI-capable steps.

---

## Deterministic validation

AI output is validated by deterministic scripts.

The validation layer checks:

- exact source-row coverage
- missing or duplicate row references
- sample-map consistency
- ChIP target/control relationships
- review flags and confidence fields
- repair audit trails

This separation is central to the trust model:

AI suggests. Deterministic code validates. Curators decide.

## Curator-facing release

The final release is generated by:

    python scripts/02_create_clean_final_release.py
    python scripts/03_qc_final_release.py

Generated output:

    results/final_curator_release/
    results/final_curator_release_<timestamp>.zip

The release contains:

    RNA/RNA_curator_review.xlsx
    RNA/RNA_AI_STUDY_SUMMARIES_CLEAN.md

## Golden-output regression checks

The current known-good artifact-backed output counts are:

    RNA study summaries: 69
    ChIP study summaries: 42
    ChIP rowwise review rows: 733
    ChIP target-control map rows: 490

These checks ensure that refactoring does not silently change the current production release.

They are regression checks, not biological claims.

## Recommended review path

1. Read README.md.
2. Read docs/ACTIVE_WORKFLOW_MAP.md.
3. Run fresh-clone smoke checks.
4. Inspect workflows/steps.tsv.
5. Review docs/GOLDEN_OUTPUTS.md.
6. Review docs/CURATOR_GUIDE.md.
7. On a production machine with generated artifacts, run artifact-backed checks.

---

## Known limitations

The repository is now production-organized and validated against existing generated artifacts, but a full controlled end-to-end rerun from raw/local inputs remains the next validation phase.

The next phase should test:

- deterministic rerun without AI
- packet regeneration
- release regeneration
- comparison to current golden outputs
- optional AI execution on a small controlled subset
